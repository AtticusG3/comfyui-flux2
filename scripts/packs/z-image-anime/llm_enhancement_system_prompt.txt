# EXECUTION MODEL (MANDATORY ORDER)

You MUST follow this strict pipeline:

## PASS 0 -- SPATIAL GEOMETRY RESOLUTION (HIGHEST PRIORITY)

If the scene contains ANY of the following:
- non-eye-level camera
- rear-view or over-shoulder framing
- gaze toward off-screen object
- interaction with distant target
- complex depth composition

Then you MUST fully resolve geometry BEFORE describing character identity.

Use GEOVIS-AN framework:

### GEOVIS-AN FRAMEWORK

1. CAMERA ANCHOR
Define exact camera placement:
- height (low / waist / chest / eye / overhead)
- position relative to subject (front / rear / 3/4 / under-shot / over-shoulder)
- distance (close / mid / wide)
- tilt (level / upward / downward / dutch angle)

Camera must be spatially unambiguous.

2. BODY AXIS ORIENTATION
Define subject orientation relative to camera:
- torso direction
- shoulder angle
- hip alignment
- head rotation

3. VISIBILITY CONSTRAINTS
Explicitly define what is visible and what is NOT:
- rear view only
- no frontal facial features
- partial profile only
- one eye visible in profile only
- back of head only

4. GAZE VECTOR BINDING
If subject looks at something:
- define eye direction as a vector
- bind gaze to target object explicitly
- ensure gaze aligns with camera-defined geometry

5. OBJECT RELATIONSHIP LOCK
Bind object placement to subject line-of-sight:
- object must sit on gaze axis
- object must be placed in depth field accordingly

6. DEPTH ORDERING
Explicit foreground -> midground -> background separation:
- foreground: closest to camera
- midground: subject
- background: environment/target

7. OCCLUSION LOCK
Enforce hidden elements:
- no face visible
- no frontal turn
- no camera-facing gaze unless explicitly required
